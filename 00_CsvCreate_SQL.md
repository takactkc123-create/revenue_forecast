# 個人住民税 課税データ抽出SQL

## 概要
Microsoft Access上で基幹システムからRDBMSを用いて課税データテーブルを抽出するSQL
個人IDは仮名化処理をしたのち、出力したCSVをPythonにインポートして分析する

## 使用環境
- Office : Microsoft Access (Microsoft Office 365)
- RDBMS  : Oracre Data Base 
- SQL    : SQL Server

## 処理内容
| 項目 | 詳細 |
|---|---|
| 対象年度　　| 2020 年 ~ 2025 年　（各年度ごとに出力）|
| 更正日　　　| 各年6月30日時点の最新データ|
| レコード　　| 1個人1レコード。更正日が同じレコードが複数ある場合は履歴番号の最大のみ抽出|
| 個人データ　| 個人を特定するIDを未公開の四則演算で仮変換。csv 出力後、pythonにて各年度の個人IDで結合|
| 年齢　　　　| 賦課期日（各年1月1日）時点の年齢を生年月日から計算し出力|

## 注意事項
- テーブル名・フィールド名は実務のものをサンプル用に置き換え済み
- 変換式の実際の数値は非公開
- SELECTの所得・控除フィールドは確定申告書第１表（令和7年分様式）の記載順に準拠
- `特定親族特別控除` は令和7年分（2025年分所得）から新設。2025年度以前の年度では該当なし

## SQL

```sql
SELECT
	-- 個人属性:年齢は賦課期日（1月1日時点の年齢を生年月日より算出）
	
    FORMAT(CLng(A.個人ID) * [乗数] + [加算値], "0000000000") AS 仮ID,
    A.性別,
    INT((CLng(20250101) - CLng(A.生年月日)) / 10000) AS 年齢,

    -- 所得金額等（確定申告書第１表 1～12の順）
    -- 給与・年金については収入から所得が自動的に決定するため、収入フィールドも追加
    -- 分離所得については、前年以前の損失に対する繰越控除後の所得をselect(税額の算出のブレを軽減する目的)
    S.事業所得_営業等,
    S.事業所得_農業,
    S.不動産所得,
    S.利子所得,
    S.配当所得,
    S.給与収入,
    S.給与所得,
    S.雑収入_公的年金等,
    S.雑所得_公的年金等,
    S.雑所得_業務,
    S.雑所得_その他,
    S.総合短期譲渡所得,
    S.総合長期譲渡所得,
    S.一時所得,
    S.総所得金額等,
    S.分離_短期譲渡一般,
    S.分離_短期譲渡軽減,
    S.分離_長期譲渡一般,
    S.分離_長期譲渡特定,
    S.分離_長期譲渡軽減,
    S.分離_一般株式譲渡,
    S.分離_特定株式譲渡,
    S.分離_上場株式配当,
    S.分離_先物取引,
    S.分離_山林,

    -- 所得控除：所得から差し引かれる金額（確定申告書第１表 13～30の順）
    K.社会保険料控除,
    K.小規模企業共済等掛金控除,
    K.生命保険料控除,
    K.地震保険料控除,
    K.寡婦控除,
    K.ひとり親控除,
    K.勤労学生控除,
    K.障害者控除,
    K.配偶者控除,
    K.配偶者特別控除,
    K.扶養控除,
    K.特定親族特別控除,
    K.基礎控除,
    K.所得控除合計,
    K.雑損控除,
    K.医療費控除,
    K.寄附金控除,
    K.差引所得控除合計,

    -- 税額控除：課税標準額（所得－所得控除）から税率をかけた後の算出所得割税額から控除する金額

    K.調整控除,
    K.配当控除,
    K.住宅借入金特別控除,
    K.寄附金税額控除,
    K.外国税額控除,
    K.配当割額・株式等譲渡所得割額の控除,
    
    -- 税額データ：市町村民税、都道府県民税、森林環境税、年税額等を抽出
    -- 目的変数として使用するのは「年税額」。均等割、所得割については、csv出力後の分布を確認することには有効だが、説明変数として用いると、その合計額＝年税額（目的変数）としてリークしてしまうため、学習用ではdropする。
    
    Z.市町村_均等割,
    Z.市町村_所得割,
    Z.都道府県_均等割,
    Z.都道府県_所得割,
    Z.控除不足額,
    Z.年税額

FROM 所得テーブル AS S

INNER JOIN 控除テーブル AS K
    ON  S.個人ID   = K.個人ID
    AND S.年度     = K.年度
    AND S.更正日   = K.更正日
    AND S.履歴番号 = K.履歴番号
    
INNER JOIN 税額テーブル AS Z
    ON  S.個人ID   = Z.個人ID
    AND S.年度     = Z.年度
    AND S.更正日   = Z.更正日
    AND S.履歴番号 = Z.履歴番号

INNER JOIN 宛名テーブル AS A
    ON S.個人ID = A.個人ID

-- 抽出条件：年度および特定の更正日を指定。更正日以前の最新履歴を抽出する。（1個人IDにつき1レコード）

WHERE S.年度 = 2025
  AND S.更正日 = (
      SELECT MAX(S2.更正日)
      FROM 所得テーブル AS S2
      WHERE S2.個人ID = S.個人ID
        AND S2.年度 = 2025
        AND S2.更正日 <= 20250630
  )
  AND S.履歴番号 = (
      SELECT MAX(S3.履歴番号)
      FROM 所得テーブル AS S3
      WHERE S3.個人ID = S.個人ID
        AND S3.年度 = 2025
        AND S3.更正日 = S.更正日
  );
```

## 逆変換クエリ（Access内管理用）

```sql
SELECT
    FORMAT((CLng(仮名ID) - [加算値]) / [乗数], "0000000000") AS 元ID
FROM 変換後テーブル
```
